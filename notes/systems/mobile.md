# Mobile target (`targets/mobile/`) — Android via Expo / React Native

The **native Android app**, built with **Expo (SDK 54) / React Native (New Architecture)**, reusing the
**shared engine** unchanged (not a WebView wrapper, not a native re-port). It ships the full four-tab app —
**Generate / Gallery / Single / Manage** — backed by on-device storage, at the app's max-load bar (100k
gallery images · up to 1000 prompt rolls · 100k-line Manage editor). **No NSFW and no local SD providers**
on the Play build (all-ages / "Everyone"); an NSFW-capable APK is GitHub-only.

## Engine reuse — the third loader

The engine is isomorphic over three loaders: `nodeLoader` (fs + `createRequire`), `browserLoader` (Vite
glob), and **`metroLoader`** (`engine/core/metroLoader.js`) — synchronous reads from a **build-time static
catalog** (`engine/core/metroCatalogData.js`, generated by `scripts/build-metro-catalog.mjs`, gitignored).
Metro can't cross the repo-root ESM package boundary, so `metro.config.js` aliases `engine` →
`../../engine` via `resolver.extraNodeModules` (not a junction, not a copy). The catalog has two tiers:
`--tier=full` and `--tier=sfw` (`catalog:play`, physically excludes nsfw-token content). Parity is proven
by `scripts/metro-parity-check.mjs` (metroLoader output == nodeLoader).

## UI — FULL parity with the web is a STANDING INSTRUCTION

**Keep the mobile UI and the web UI in COMPLETE parity — no exceptions, no size-based feature loss, no
"mobile is simpler" version.** The whole app is available on every device (phone AND tablet) **to the extent
the target platform allows**, and parity is **enforced behind the release quality gate**. The only sanctioned
non-parity is a genuine platform boundary expressed as a **build variant, not a feature drop** — the SFW
build physically removes NSFW (content + toggle) via `--tier=sfw`; an NSFW-capable APK keeps it. See the
campaign + gap audit + phased plan in [`../plans/mobile-parity.md`](../plans/mobile-parity.md). This is
bidirectional: a change to one surface's look/layout/features must be reflected in the other. Concretely the
mobile app mirrors:

- **Top bar** (`App.js`): dark rounded brand mark (mint pencil) + the **enclosed pill view-switch** (active
  tab green-filled) + a `⋯` overflow — one row, matching the web phone header. Panes stay mounted
  (visibility toggled) so each keeps its state/scroll across switches, like `App.jsx`.
- **Generate** (`screens/GenerateScreen.js`): the web `PromptComposer` — an **Insert ▾** dropdown, a
  code-editor prompt box (line-number gutter + `{…}` syntax highlighting via a highlight layer under a
  transparent `TextInput`, a ✓ valid indicator + eye/gear icons), a **PROMPTS** count, a tool toolbar
  (wand · tag · brackets[active] · share · shuffle) + a round green **generate** button, and the bottom-left
  green **building-blocks FAB** (opens the palette bottom sheet — the web's off-canvas drawer). Results are a
  virtualized FlashList (memoized rows, stable ids) to the 1000-prompt max.
- **Gallery / Single / Manage**: recycling FlashList of uniform cells (100k) · expo-image detail ·
  master/detail windowed 100k-line editor (uncontrolled per-line inputs → no list re-render on keystroke).

**Design tokens** (`lib/theme.js`) mirror `styles/foundation/tokens.css` (mint `#34e2a0` accent + dark ink,
`#1c1c1f` canvas, `#232328` panels, 14px radius). **Icons** (`lib/icons.js`) are hand-written
`react-native-svg` line-icons matched to the web's icon set (Expo Go bundles react-native-svg, so no native
rebuild). Storage (`lib/storage.js`) is expo-file-system, web-safe no-op under `Platform.OS === 'web'`.

## Build / run / verify

- `npm run catalog` (full) / `npm run catalog:play` (sfw) — regenerate the Metro catalog (prestart hook).
- `npm start` → `expo start` (add `--tunnel` for on-device Expo Go over ngrok; LAN otherwise).
- **Visual verification in CI/cloud**: `expo export --platform web` (react-native-web) → serve the
  `dist-web/` bundle → Playwright screenshot at 390px, compared against a web reference render. `dist-web/`
  and run logs are gitignored.

## Testing

Four layers, mirroring the web target's rigor:

- **Engine/data parity** — `scripts/metro-parity-check.mjs` (metroLoader output == nodeLoader).
- **Ported-catalog + surface parity** — `scripts/mobile-parity-check.mjs` (accents/locales/DPL/providers
  match the web SOURCES, and each focus surface — Header / Generate / Gallery / Single — carries a marker
  for every web feature; no per-feature ignores). Wired into `npm test`.
- **Component/behavior** — `jest-expo` + React Native Testing Library mount the REAL screens (the Android
  code path) with native modules mocked (`targets/mobile/jest.setup.js`) and assert render + wiring. Run
  `npm --prefix targets/mobile test` (root `test:mobile`, in `npm test`). Pure helpers (`lib/single.js`)
  also have root-vitest unit tests under `tests/mobile/`.
- **Visual/UX parity** — `expo export --platform web` → Playwright screenshot at 390px vs a web reference.
